\chapter*{摘\quad 要}
\addcontentsline{toc}{chapter}{摘要}

本文设计并实现了一套基于 HarmonyOS 的听障人士交互训练系统 HearBridge，旨在为听障用户及手语学习者提供课程学习、手势训练、实时识别、句子视频识别、训练反馈和资源管理等功能支持。系统围绕听障辅助交流与训练场景展开设计，构建了由 HarmonyOS 移动端、Spring Boot 服务端、Vue 后台管理端和 Python 手势识别服务组成的多端协同应用体系。移动端面向普通用户提供课程浏览、数字人动作示范、手势训练、训练结果查看和句子视频识别结果展示等功能；服务端负责用户认证、课程与训练数据管理、文件资源管理、识别服务调用和智能语义处理；后台管理端负责手势分类、手势资源、样本数据、训练任务和模型版本的维护；Python 识别服务则承担关键点提取、特征转换、模型训练、实时单词识别和句子视频离线识别等任务。

在系统实现方面，本文将用户侧训练流程与管理侧模型维护流程进行相对分离。用户侧以“标准动作展示—用户模仿训练—识别结果反馈”为主要流程，通过数字人资源为用户提供动作示范，并通过实时识别或视频识别能力对训练结果进行辅助评估；管理侧则通过样本管理、特征转换、模型训练、模型版本发布等功能，为识别模型的更新和维护提供支撑。针对手势识别任务，系统采用 MediaPipe 提取手部和姿态关键点，并基于 TensorFlow 构建固定词表下的时序分类模型，实现对手语动作的识别。在此基础上，本文进一步实现了面向有限词表的手语句子视频识别链路：系统接收用户上传的手语句子视频后，由 Python 识别服务进行动作窗口分析、词级候选序列生成和中文映射，再由 Spring Boot 服务端调用大语言模型进行语义修正与结果组织，从而生成更便于用户理解的句子级识别结果。

测试与实验结果表明，HearBridge 系统能够稳定完成用户登录、课程浏览、手势训练、实时识别、视频上传识别、后台资源管理、样本管理、模型训练和模型版本发布等核心功能。在有限词表实验条件下，句子视频识别模块能够完成从视频输入到词序列输出、中文展示和语义修正的完整流程，验证了将手势识别模型与语义后处理方法结合用于辅助交流场景的可行性。本文实现的系统为听障辅助训练应用提供了一个可运行的工程原型，也为后续扩展连续手语识别、手语辅助翻译、数字人联动展示和个性化训练反馈等功能奠定了基础。
