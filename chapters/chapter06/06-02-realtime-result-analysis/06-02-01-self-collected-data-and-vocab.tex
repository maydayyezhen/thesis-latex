\subsection{自采数据规模与词表说明}

实时单词识别模型使用 HearBridge 系统自采集的手势样本进行训练。

该部分数据来源于移动端训练页面中的样本采集流程，
与后台管理端维护的手势资源和训练动作对应。
移动端在采集过程中持续向 Python 手势识别服务发送图像帧，
Python 服务对图像帧进行 MediaPipe 手部与姿态关键点检测，
并将满足有效帧条件的结果保存为 raw dataset 样本。

与公开手语数据集相比，
自采数据规模较小，
但它能够更直接地验证 HearBridge 系统自身的数据采集、特征转换、模型训练和模型发布链路。
本次实时单词识别实验共使用 80 个自采特征样本，
其中训练样本 64 个，
验证样本 16 个，
支持类别数为 8 类。
每个样本被转换为 $30\times166$ 的时序特征，
其中 30 表示固定时间窗口内的帧数，
166 表示单帧关键点与姿态相关特征维度。

本次训练所使用的固定词表包括数字类标签和颜色类标签。
其中，
数字类标签包括 \texttt{0}、\texttt{1}、\texttt{2}，
颜色类标签包括 \texttt{black}、\texttt{blue}、\texttt{brown}、\texttt{colour} 和 \texttt{cream}。

从词表构成可以看出，
本次实时单词识别实验并不追求覆盖大规模开放词汇，
而是优先验证系统在小规模固定词表下的识别闭环是否可行。

这种设计适合毕业设计阶段的系统验证。
一方面，
小词表能够降低样本采集和模型训练成本，
便于快速观察模型训练效果；
另一方面，
这些标签能够覆盖不同类型的简单手势，
可以用于检验关键点提取、特征转换、分类推理和移动端结果展示等流程是否能够正常工作。

总体来看，
自采数据集的价值不只在于训练一个固定词表分类模型，
更重要的是验证 HearBridge 系统的数据闭环。
系统能够从移动端完成样本采集，
在 Python 服务中进行关键点提取和特征转换，
再通过训练流程生成模型文件和标签映射文件，
最终由后台管理端发布模型版本并接入移动端实时识别。
这一闭环为后续扩充词表、增加样本规模和持续优化识别模型提供了基础。
